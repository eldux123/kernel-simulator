# ===================================================================
# ANÁLISIS Y CONCLUSIONES:
# ===================================================================
#
# 1. FCFS (First Come First Served):
#    - Ventajas: Simple, justo, fácil de implementar
#    - Desventajas: Alto movimiento total, no optimiza recorrido
#    - Uso recomendado: Sistemas con baja carga de disco
#
# 2. SSTF (Shortest Seek Time First):
#    - Ventajas: Mínimo movimiento local, muy eficiente
#    - Desventajas: Puede causar starvation en cilindros extremos
#    - Uso recomendado: Sistemas con alta carga, accesos distribuidos
#
# 3. SCAN (Elevador):
#    - Ventajas: Balance entre eficiencia y justicia, predecible
#    - Desventajas: Más movimiento que SSTF, pero previene starvation
#    - Uso recomendado: Sistemas multiusuario, cargas variables
#
# TRADE-OFFS:
# - Rendimiento puro: SSTF > SCAN > FCFS
# - Justicia/Fairness: FCFS = SCAN > SSTF
# - Predecibilidad: SCAN > FCFS > SSTF
# - Complejidad: FCFS < SSTF < SCAN
#
# RECOMENDACIÓN:
# - Uso general: SCAN (mejor balance rendimiento/justicia)
# - Alto rendimiento: SSTF (si no hay problema de starvation)
# - Sistemas críticos: FCFS (máxima predecibilidad)
# ===================================================================
